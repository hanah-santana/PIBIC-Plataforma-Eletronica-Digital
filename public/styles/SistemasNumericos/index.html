Sistemas Numéricos

Sabe-se que, de maneira diferente do cotidiano, os processamentos e aparelhos digitais utilizam sistemas binários, hexadecimais ou octais de numeração para obterem um melhor desempenho. Desta maneira, não se limitam somente ao sistema decimal, como na primeira geração de computadores que utilizavam válvulas termoiônicas. Para a melhor compreensão da lógica computacional e de sistemas afins, será feito o estudo dos sistemas de numeração.

Sistemas de Numeração


Decimal

O sistema decimal possui 10 dígitos (0-9) e é constituído pelas unidades, dezenas, centenas, milhares etc. Vejamos sua representação no exemplo a seguir:

49010 = 0x100 + 9x101 + 4x102
129410 = 4x100 + 9x101 + 2x102 + 1x103

Binário

É o sistema de numeração que possui 2 dígitos (0 e 1) e é largamente utilizado em sistemas lógicos que normalmente usam álgebra booleana, tais como sistemas computacionais e afins. Será representado de maneira que seja convertido para decimal a seguir:

102 = 0x20 + 1x21 = 0 + 2 = 210
10112 = 1x20 + 1x21 + 0x22 + 1x23 = 1 + 2 + 0 + 8 = 1110

Hexadecimal

Este sistema é representado por 16 dígitos (0-F). Desses 16 dígitos, 10 são conhecidos por fazerem parte do sistema decimal, acrescidos de 6 dígitos, que são A, B, C, D, E, F. Comparando ao sistema decimal, tem-se que A=10, B=11, C=12, D=13, E=14, F=15

AB16 = 11x160 + 10x161 = 11 + 160 = 17110
34916 = 9x160 + 4x161 + 3x162 = 9 + 64 + 768 = 84110
DA316 = 3x160 + 10x161 + 13x162= 3 + 160 + 3328 = 349110
1C16 = 12x160 + 1x161 = 12 + 16 = 2810

Octal

Este sistema já foi utilizado como uma alternativa mais compacta em relação ao sistema binário para programação em linguagem de máquina. Possui 8 dígitos compreendidos entre 0-7.

728= 2x80 + 7x81 = 2 + 56 = 5810
4538 = 3x80 + 5x81 + 4x82 = 3 + 40 + 256 = 29910

Outros Sistemas

É importante saber que os sistemas numéricos não limitam-se apenas aos binários, hexadecimal e afins. Na realidade, pode-se utilizar qualquer base numérica e realizar operações, seja com 3, 4, 9, 12 ou qualquer outro número de dígitos. Estes não serão vistos aqui por não serem utilizados.

Conversões

Ao iniciar os estudos na computação, torna-se comum lidar com números em diferentes sistemas numéricos, tais como Hexadecimal e Binário. Com isso, atribuir valores presentes em uma base a outra, ou seja, converter números em diferentes bases numéricas, mostra-se necessário.

Conversão de Outras Bases Para Decimal:

Como visto na seção anterior, que descreve cada sistema de numeração, é possível efetuar a conversão de qualquer base para a base 10 (decimal), utilizando-se do mesmo método. Vejamos um exemplo simples:

Exemplo: Digamos que você está tentando converter A3B16 para decimal. Primeiro selecione o último dígito (B, que equivale à 11) e multiplique-o por 160(estamos multiplicando por 16 porque a base do número A3B é a hexadecimal), some com o penúltimo dígito (3) multiplicado por 161 e, finalmente some com o antepenúltimo dígito (ou primeiro, A=10) multiplicado por 162.

Teremos: A3B16 = 11x160 + 3x161 + 10x162 = 261910

Existem vários outros exemplos deste tipo de conversão na parte de Sistemas de Numeração, onde estão as representações em decimal de todos os sistemas numéricos supracitados, logo acima nesta página.

Passo a Passo:

Temos então o método passo a passo para facilitar a memorização. Lembrando que este método serve para converter de todos os sistemas numéricos (binário (base 2), hexadecimal (base 16), octal (base 8)... etc) para decimal (base 10). Vejamos:

1. Cada dígito do número a ser convertido para decimal, deve ser multiplicado pelo número correspondente a base na qual o número pertence. Por exemplo, se você está tentando converter da base hexadecimal para a decimal, multiplique cada dígito do número a ser convertido por 16. Se for da binária para decimal, multiplique cada dígito por 2. Se octal, por 8, e assim por diante.

2. Iniciando pelo último dígito do número a ser convertido, multiplique pelo número da base (correspondente ao passo anterior) elevado a 0. O penúltimo multiplique pelo número da base elevado a 1, o antepenúltimo deve ser multiplicado pela base elevada a 2, depois por 3… e assim por diante.

3. Depois faça a soma de todos os resultados das operações anteriores.

4. Caso o sistema seja hexadecimal ou afins, utilizar os valores correspondentes aos dígitos, para que seja possível calcular (por exemplo: A=10, B=11...).

Vale lembrar que existem vários exemplos na seção anterior.

Exemplo 2 (Decimal - Hexadecimal):

Outras Conversões

O processo de conversão de decimal para qualquer outra base tornou-se simples. O diferencial é o divisor que será utilizado. Nota-se que é correspondente ao sistema decimal para o qual o número será convertido.

Caso seja necessário fazer uma conversão de Binário para Hexadecimal, por exemplo, você pode utilizar os métodos acima para converter Binário para Decimal, e logo depois, converter de Decimal para Hexadecimal.

Operações Aritméticas

Ao executar operações básicas de aritmética como Adição e Subtração, são utilizados métodos simples vistos desde o ensino fundamental. Estes métodos, mesmo que muitos não saibam, podem ser aplicadas à diferentes bases se adaptando, claro, aos algarismos disponíveis dependendo do sistema numérico em questão. Vejamos:

Soma:

Ao executar uma soma no sistema decimal, com os números 28 e 4 por exemplo, prossegue-se da seguinte maneira:

1. Deve-se “armar” a conta, alinhando as parcelas a serem somadas. Unidade, acima de unidade, dezena acima de dezena e assim por diante.

2. A soma de cada parcela é feita da direita para a esquerda.

3. O importante, e que varia dependendo da base, é justamente esta soma. Ao somar 8 e 4, o sistema decimal não possui algarismos suficientes para representar o resultado.
(Já no sistema hexadecimal, por exemplo, teríamos o B. 9+3 = 1210, 1210 = B16)

4. Portanto, ao atingir o maior algarismo do sistema numérico, que no sistema decimal é o 9, deve-se “reiniciar” a contagem no menos significativo indo para 0, e continuar a contagem a partir dele.

5. Feito isto, deve ser adicionado 1 ao mais significativo logo a direita, tal como é feito em uma soma comum.

Estas operações podem se repetir para todas as parcelas da soma e ser executadas da mesma maneira para todas as bases numéricas.

Vejamos alguns exemplos:

Subtração:

Ao executar a operação de subtração, o método utilizado que também já é conhecido desde o ensino fundamental, é simples e útil para todos os outros sistemas de numeração além do decimal. Vejamos primeiro casos simples, onde a diferença entre os números é pequena:

Basta observar que as unidades são subtraídas normalmente, de maneira independente do sistema numérico em questão.

Na operação subtração, por vezes, nos deparamos com o fato de alguma(s) da(s) parcelas do subtraendo ser menor que a do minuendo. Com isso, torna-se necessário “pedir emprestado” ao algarismo mais significativo ao lado.
Com um exemplo, recaptularemos primeiro como se dá este processo no sistema decimal:

Demonstraremos o funcionamento uma simples subtração 31-9=21. E a partir disso, entender como esta funciona para os demais sistemas numéricos.

1. Nota-se que, na parcela das unidades, o Subtraendo é maior que o Minuendo. O que acaba sendo um empecilho para executar a operação.
Dessa forma, retira-se uma dezena para transferir para as unidades.

2. Ao retirar 1 das dezenas, transferimos 10 unidades e somamos com as unidades existentes no minuendo. Isso se deve ao fato de, no sistema decimal, uma dezena ser composta por 10 unidades, uma centena ser composta por 10 dezenas e assim por diante.

3. Então, de maneira simples, temos que: Ao "pedir emprestado" deve-se retirar um do mais significativo e adicionar 10 (porque estamos no sistema decimal) ao menos significativo.
Feito isto, o resultado é extraído normalmente, como nos casos mais simples.

Para consolidar o entendimento, vejamos o funcionamento deste processo em outras bases numéricas. Nisso, nos deparamos com duas maneiras diferentes, porém parecidas de fazê-lo. Veremos ambas.

Como exemplo, o sistema octal, método 1:
Este método segue exatamente o que é feito no método descrito acima, vejamos:

1. Observa-se que, na parcela menos significativa da subtração, o Subtraendo é maior que Minuendo. Desta forma, é necessário "pedir emprestado" do mais significativo para seguir com a subtração.

2. Relembrando o que foi dito alguns tópicos atrás: "Ao retirar 1 das dezenas, transferimos 10 unidades e somamos com as unidades existentes no minuendo."
Neste caso, deve-se adicionar 8 unidades ao menos significativo, uma vez que a operação é feita no sistema octal. Transferindo assim, 1 "octeto"(8, que é representado por 10 no sistema octal) para as unidades.

3. É importante lembrar que, uma vez que o sistema octal vai de 0 a 7, a operação de soma 10+2=12 e de subtração 6-10, no sistema octal, resulta em 4.

4. Feito isto, a operação pode ser concluída normalmente. (10-6=4; 3-1=2 e, finalmente, temos que: 42-16=24)

Utilizando o mesmo exemplo, método 2:Neste método, algumas das operações internas para chegar ao resultado final utilizam o sistema decimal para facilitar a compreensão.

1. Iniciando da mesma maneira, nota-se que o subtraendo é maior que o minuendo (nas unidades), e deve-se fazer o processo de "pedir emprestado".

2. Da mesma maneira, transferiremos um "octeto" para as unidades, mas ao invés de representá-lo por 108, o "octeto" será representado pelo valor no sistema decimal, ou seja, 810. Sendo assim, também serão transferidas 8 unidades para o menos significativo.

3. Feito isto, serão somados 8 e 2 (no sistema decimal), obtendo 1010.

4. Desta maneira, a subtração pode ser finalizada normalmente. Atentando-se ao fato de que as somas(como visto no tópico 3, acima) e as subtrações(como vistas logo abaixo 10-6=4) são feitas no sistema decimal.
